Change tracker

The change tracker application CT answers some questions about how some software under test SUT has evolved over a span of time. The start and end of that span of time are described by two sets of commit specifications. A commit specification, or cspec, identifies a source control repo and commit ID from that repo. A cspec set, or set of cspecs, describes a moment in the source control history of an SUT spanning multiple source control repositories. Cspec sets are described by JSON, or by a URL to a JSON file on a web server, or by a hybrid of JSON containing URLs pointing at further JSON (potentially going many levels before fully resolving). See the simple CT UI to experiment with different inputs.

Inputs

  • operation
    • list_bug_IDs_between
    • list_changes_between
    • list_files_changed_between
  • starting cspec set
  • ending cspec set
